# AfghanTours — Attractions Instructions (Strict Enforcement)

## Purpose
Defines how attractions should be structured across:
- main attractions page
- attraction detail pages
- attraction drawers
- cross-linking with tours and hubs

---

## Attraction Philosophy
Attractions are a core experiential layer, not just content pages.

They must connect to:
- hubs
- provinces
- routes
- tours
- map interactions

---

## Main Attractions Page
The attractions index page should:
- feel premium and image-led
- allow browsing by region / province / category
- stay scannable and not text-heavy

Each attraction card should include:
- image
- name
- province
- category
- teaser
- optional access cue

Do not overload the cards with operational detail.

---

## Attraction Detail Page Structure
Required sections:
1. Hero image
2. Breadcrumb
3. Attraction title
4. Intro summary
5. Main description
6. Access / practical notes
7. Nearby hub
8. Related tours
9. Optional gallery
10. Optional map context

---

## Attraction Drawer
The attraction drawer is mandatory in itinerary interactions where attraction chips are used.

It should:
- open without navigating away
- show image, name, location, summary, and access context
- support a link to the full attraction page

Do not break the drawer while revising itinerary logic.

---

## Content Rules
- Use exact attraction names from data.
- Do not invent historical claims.
- Do not imply easy access when access is actually remote.
- If entry fees exist in data, show them carefully and accurately.
- If an attraction is inactive, do not surface it in public-facing pages.

---

## Expedition / Remote Rules
If an attraction is remote or expedition-linked:
- distinguish that from standard hub-daytrip access
- do not present it as a casual stop

---

## Related Tours
If tours include the attraction:
- surface them visibly
- use cards or chips
- do not bury them in long text

---

## Styling Rules
Attractions must feel:
- visual
- premium
- clear
- modular

Do not let attraction pages become raw database dumps.
